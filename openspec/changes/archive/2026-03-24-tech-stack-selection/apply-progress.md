## Issues Encountered

### Issue 1: Vite scaffold removed openspec directory
- **Problem**: `create-vite@5 .` with "Remove existing files" option deleted `openspec/` directory
- **Resolution**: Recreated `openspec/` structure after scaffolding

### Issue 2: shadcn v4 interactive-only CLI
- **Problem**: `npx shadcn@latest init` (v4.0.8) changed to interactive-only; no `--yes --defaults` for Vite+JS
- **Resolution**: Manually created `components.json`, `src/lib/utils.js`, and `src/components/ui/button.jsx`
- **Spec note**: Spec task 3.5 explicitly permits manual creation

### Issue 3: JSX in .js test files not transforming
- **Problem**: Vitest v1 uses SSR transform mode for Node.js environments; `@vitejs/plugin-react` skips Babel JSX transform in production/SSR mode for `.js` files (it only applies to `.jsx` files or when in fast-refresh mode)
- **Root cause**: `canSkipBabel()` returns true in production → React plugin transform deleted → esbuild also doesn't transform JSX in `.js` → `ssrTransformScript` fails
- **Resolution**: Added `babel: { presets: ['@babel/preset-react'] }` to the React plugin config. This forces Babel to run and transform JSX even in SSR mode.
- **Impact**: `@babel/preset-react` added as devDependency

### Issue 4: eslint.config.js conflict
- **Problem**: Vite scaffold generated `eslint.config.js` (flat config for ESLint v9), conflicting with our `eslint@8` and `.eslintrc.cjs`
- **Resolution**: Deleted `eslint.config.js`

### Issue 5: ESLint vitest globals
- **Problem**: `describe`, `it`, `expect` reported as undefined by ESLint (vitest globals not known)
- **Resolution**: Added `overrides` to `.eslintrc.cjs` with `env.jest: true` for test files (vitest globals are identical to jest globals)
